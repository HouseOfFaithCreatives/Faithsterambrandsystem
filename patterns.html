03 — Visual Identity

Motifs & Imagery

A graphic language layer the current brand has none of — reusable motifs separate from the logo, a defined photography direction, and a genre/category color system, since none of these exist anywhere in the current guide or the live product.

New — no existing precedent

Graphic motifs

Drawing on the brand essence rather than generic streaming-platform gradients — light and radiance (tied to faith imagery), woven pattern (tied to African textile traditions), and a subtle dot/halftone texture for backgrounds that need quiet visual interest without competing with content thumbnails.

Usage discipline
All three motifs follow the same rule as Netflix and UNC's pattern systems: subtle, used as texture or background layer — never as the dominant visual element, never directly behind body copy, never used to construct a custom logo.

Bug found in the live product, not just this document
The current FaithStream code uses tv-2, music-4, and mic-2 as Lucide icon names in the nav drawer (Series, Worship, Podcasts). Current Lucide has renamed these — the correct names are tv-minimal, music, and mic. This means those three nav icons are likely rendering blank in the live app right now. Worth a one-line fix in the product code itself, independent of this brand system.

Rule
Outline icons throughout — never switch a single icon to a filled style except the play triangle and similar primary-action glyphs, which read faster filled. Mixing outline and filled icons in the same row (e.g. nav bar) is the most common, most visible iconography mistake to avoid.

Motion & transition language

The live product already has a considered motion system for component interactions — these are the actual timing/easing values in use, documented here so new components match instead of guessing.

InteractionTimingEasingWhere it's used
Card hover lift0.28scubic-bezier(.4,0,.2,1)card-land, card-port, card-cont
Hero slide transition0.65scubic-bezier(.4,0,.2,1)hero-track
Nav drawer slide-in0.32scubic-bezier(.4,0,.2,1)nav-drawer
Modal entrance0.3scubic-bezier(.22,1,.36,1)modal-box (slight overshoot — more "premium" feel)
Hover preview popup380ms delay + 0.22seasepreview-popup (delay prevents accidental triggers while scanning)
Rule
cubic-bezier(.4,0,.2,1) (Material-style "standard" easing) is the default for anything functional — hovers, drawers, slides. Reserve cubic-bezier(.22,1,.36,1) (slight overshoot) for moments that should feel a little more special, like the full info modal opening — don't use it everywhere or it stops feeling special.

Five animated background directions

Five pure-CSS motion backgrounds, no video dependency — usable directly on hero sections, loading screens, and splash states while your own motion video is in production, or as a lighter-weight permanent alternative to a rendered video loop.

FaithStream
01 — Aurora Glow
Two soft blurred light masses drift slowly behind the logo — calm, ambient, faith-forward. Best for splash screens and donation pages.
FaithStream
02 — Mesh Gradient Drift
The full purple range flowing slowly through itself. Most "premium streaming platform" of the five — good for the main hero background.
FaithStream
03 — Ember Drift
Small magenta embers rising slowly, tied to the "radiance" motif from this same page. Distinctive and a little more playful — good for loading states.
FaithStream
04 — Radial Pulse
A slow breathing glow from center — the calmest option, good for a "waiting" or buffering state where you don't want to add visual noise.
FaithStream
05 — Diagonal Sweep
A soft silk-like light band moving across the frame. The most cinematic and "key art" feeling of the five — good for marketing hero sections.
Recommendation
Mesh Gradient Drift (02) is the closest match to the existing product's hero gradients already in use — recommend it as the default if only one of these five gets adopted permanently, with Aurora Glow (01) as the calmer alternative for donation/give screens specifically, matching that flow's gentler tone.
