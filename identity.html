Minimum size rule

The full lockup must never appear narrower than 48px. Below that threshold the wordmark becomes unreadable even at retina resolution. Always switch to the icon-only file at compact sizes.

⚠ Minimum — 48px wide

This is the smallest the full lockup may appear in any medium. At this size the wordmark is still legible on retina displays. Anything smaller must use faithstream_icon_purple.png instead.

Co-brand lockup

When FaithStream appears alongside a partner brand, use the wordmark-only file and separate the two lockups with a 1px vertical rule. Neither brand should dominate — equal optical weight, equal vertical alignment. Minimum clear space between the separator and each logo is 24px.

Co-brand rules

Note on the wordmark typeface
The wordmark itself is set in a serif display face — the same one named "Miller Text" in House of Faith's original one-page brand guide. This system's Syne/Inter typography rules (below) govern everything around the logo — UI, headlines, body copy — not the wordmark itself. The logo is a fixed asset, not something redrawn in the new type system; retiring Miller Text from headlines elsewhere doesn't require changing the logo file.

Usage rule

Rule
Purple carries every accent role in this system. No gold, no teal, no hue outside this family — confirmed direction for FaithStream after testing both an expanded multi-hue palette and an all-purple range head to head. If a new use case (a chart, a status indicator) seems to need a color outside this range, that's a flag to bring to review, not a default to reach for.

Functional color — open question

One real exception already exists in the live code: #4ade80 (green) is used for the "Match %" indicator on hover previews and modals — a Netflix-convention color that arrived without a deliberate brand decision behind it.

OptionWhat it means
A — Retire itReplace with a purple-family equivalent (e.g. --lavender-light for match %, a warm-shifted magenta for errors). Fully purple-only system, no exceptions.
B — Keep as the one exceptionGreen stays specifically for "Match %" since it's a near-universal streaming-platform convention users already read instantly; everything else (errors, alerts) still resolves to purple-family colors.
Decision needed
Not resolved in this draft — flagging for review rather than silently picking one. Worth deciding once, then documenting the answer here so it doesn't get re-litigated per feature.

Typography System

Inter + Syne already shipped — formalizing the system around them

The live product already uses Syne for display moments and Inter for everything else — confirming the direction tested earlier in this project. This section formalizes the rule set, shows the system in real creative context, and flags one inconsistency to resolve.

Syne
Display — logo, hero titles, modal titles, Top 10 numeralsGeometric, confident, quietly architectural. Used at 700–900 weight. Already running in .topnav-logo, .modal-title, .card-port-num.
Inter Extrabold
Hero titles, primary CTAsUsed in .hero-title at weight 800 — the workhorse for anything that needs to feel urgent and legible at small sizes.
Inter Semibold / Medium
Card titles, nav, section headersThe default UI weight throughout — card-land-title, section-title, drawer items.
Inter Regular
Body copy, descriptions, metadataModal descriptions, card metadata, captions — anywhere reading comfort matters more than emphasis.
Inconsistency flagged
One hero slide (.hero-tagline, "Stepping into Maggie's Shoes") uses Cinzel, a serif face that appears nowhere else in the system — likely added for one moment of flourish rather than as a deliberate third typeface. Recommend retiring it: either drop the tagline treatment or restyle it in Syne at a lighter weight, so the type system stays to two families everywhere, the same discipline already applied to color.
