04 — Product

Components

Every component below is lifted directly from the live FaithStream product code — not redesigned, just documented and annotated against the token system, so new screens reuse these patterns instead of building slightly-different versions each time.

All components below are live product code, unchanged

Buttons

Extended — three core styles already shipped, expanded into a full system

The live product ships three button styles (primary white, secondary glass, brand plum). This expands that into a complete system — eight variants, three sizes, icon-only circular buttons, pill filters, and the states every button needs (loading, disabled, full-width) — so new screens stop inventing one-off button treatments.

Icon-only / circular

Filled circular buttons are reserved for the play action specifically (resume cards, hover previews, modals) — outline and ghost variants cover everything else (add to list, like, share) so the play button always reads as the one filled, unmistakable action in any card.

Pill filters / tags

For genre filters, language toggles, or any horizontal multi-select row — active state uses the magenta accent, matching the genre color system on the Motifs & Imagery page.

States
Full-width (mobile / forms)

Reserved for mobile CTA bars and form submissions — never used in desktop content rows, where a full-width button would compete with the card grid for visual weight.

Badges

Content metadata — rating, genre, duration — shown as small pills. The rating badge is the only one that uses the magenta accent; everything else stays neutral so it doesn't compete for attention.

Cards

Three card types, used for three distinct content rails. All three share the same hover language: scale 1.03–1.05, lift 4–5px, shadow growth — consistent enough to feel like one system.
